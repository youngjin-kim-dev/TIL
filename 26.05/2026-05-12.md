## 배운 점

### SQL — `||` vs OR
- `||` 비표준 — DB마다 의미 다름 (MySQL OR, Oracle 문자열 연결, SQL Server 에러)
- 표준 SQL은 `OR` — 어디서든 작동, 이식성 좋음
- "통과했어도 표준 풀이가 다를 수 있다" 인식

### SQL — IN의 좌우 응용
- 기본: `WHERE 컬럼 IN (값1, 값2, 값3)`
- 응용: `WHERE 값 IN (컬럼1, 컬럼2, 컬럼3)`
- 같은 값을 여러 컬럼과 비교할 때 활용

### SQL — 비트 연산 (융합 학습)
- `&` 연산자 — SQL에서 비트 AND 가능
- 비트 플래그 패턴 — 한 컬럼에 여러 boolean 저장
- `SKILL_CODE & CODE > 0` — 그 비트가 1인지 확인
- 컴퓨터 구조의 AND 연산 → SQL 응용

### SQL — 서브쿼리 개념 정리
- 쿼리 안의 쿼리 (괄호로 감쌈)
- 서브쿼리 먼저 실행 → 메인 쿼리가 그 결과 사용
- WHERE 절에서 자주 사용: `= (서브쿼리)`, `IN (서브쿼리)`
- 사실 이전 과일 아이스크림 문제에서 이미 사용했음 (모르고 사용)

### SQL — DISTINCT
- 중복 제거
- JOIN으로 같은 행이 여러 번 매칭될 때 사용

### Python — 이스케이프 문자
- `\\` — 백슬래시
- `\'` — 작은따옴표
- `\"` — 큰따옴표
- `\n`, `\t` — 줄바꿈, 탭 (이전 학습)
- `\`는 "다음 문자를 특수 처리하라" 신호

### Python — 따옴표 선택 전략
- 출력에 `'` 있으면 → `"..."`로 감싸기
- 출력에 `"` 있으면 → `'...'`로 감싸기
- 둘 다 있으면 → 이스케이프 또는 삼중 따옴표

### Python — f-string 응용
- `f'{a} + {b} = {a + b}'` — 수식도 넣을 수 있음
- `{}` 안에 — 변수, 수식, 메소드 호출 모두 가능

## 막힌 부분
- `||` 사용해서 통과는 했지만 표준 아니라는 거 인지
- 비트 연산이라는 개념 처음 — AND 연산 이미 알았지만 SQL 응용은 처음
- 서브쿼리 — 이미 사용했지만 이름·개념 정확히 학습은 오늘
- 따옴표 안에 따옴표 — 이스케이프 처리 방법 모름
