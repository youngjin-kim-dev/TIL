## 배운 점

### SQL
- LIKE 패턴의 정밀도 — `'서울%'` (시작 매칭) vs `'%서울%'` (포함 매칭)
- GROUP BY의 필수성 — 집계 함수(AVG 등) + 일반 컬럼 함께 SELECT 시 필요
- ROUND의 두 번째 인자 N — "N자리에서 반올림"과 ROUND(..., N-1)의 변환
- ORDER BY에서 SELECT 별칭 사용 가능

### Python
- 좌표의 음수 표현 — 반대 방향을 음수로 (벡터 계산 기초)
- 문자열 순회는 리스트와 문법 동일 (`for i in route`)
- 컴퓨터는 0부터, 사용자는 1부터 셈. 학생번호 - 1 = 인덱스 패턴 응용

### 자료형 의식의 중요성
- "10" (문자열) vs 10 (숫자) — 따옴표가 자료형 신호
- 자료형에 따라 같은 연산자도 동작 다름
