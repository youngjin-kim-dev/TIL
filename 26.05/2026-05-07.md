## 배운 점

### SQL — UNION 첫 학습
- JOIN vs UNION 구분
  - JOIN: 옆으로 (열 추가, 관계로 연결)
  - UNION: 위아래로 (행 추가, 같은 종류 데이터 합치기)
- UNION의 규칙
  - 위·아래 SELECT의 컬럼 개수·순서·자료형 동일
  - ORDER BY는 마지막 한 번만
- UNION vs UNION ALL — 중복 제거 vs 그대로
- 한 테이블에 컬럼 없을 때 — `NULL AS 컬럼명`으로 채우기

### Python — 리스트 메소드들
- `list.index(값)` — 값의 위치 반환
- `값 in 리스트` — 포함 여부 (True/False)
- `max()`, `min()`, `sum()`, `len()` — 내장 집계 함수
- 두 리스트로 키-값 매핑 패턴 — 같은 인덱스로 연결
  - 다음 학습: 딕셔너리(dict)가 이 패턴을 더 깔끔하게 표현

## 막힌 부분
- UNION 문제 — JOIN 패턴으로 시도했다가 막힘
  - "두 테이블 데이터 합치기" = UNION이라는 거 늦게 인식
  - JOIN과 UNION 차이 처음 명확히 학습

## 새로 학습한 패턴
- UNION으로 두 테이블 위아래 합치기
- `NULL AS 컬럼명`으로 없는 컬럼 채우기
- list.index() 메소드 사용
