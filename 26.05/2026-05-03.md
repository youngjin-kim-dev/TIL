## 배운 점

### SQL
- IFNULL 함수 — NULL 값을 대체값으로 변환 (MySQL 전용)
- COALESCE 함수 — 표준 SQL의 NULL 처리, 여러 인자 가능
- DATE_FORMAT 함수 — 날짜를 원하는 포맷 문자열로 변환
  - %Y(4자리 연도) vs %y(2자리 연도) 차이
- OR vs IN 연산자 — 여러 조건 비교 시 IN이 더 가독성 좋음
- 함수 안에서도 테이블 별칭 사용 — 두 테이블에 같은 컬럼명 있을 때
- Nullable: TRUE는 NULL 처리 필요 신호

### Python
- `+` 연산자의 자료형별 의미 — 숫자는 덧셈, 문자열은 연결
- `**` 거듭제곱 연산자 (a**2 = a의 제곱)
- `int()`로 소수점 버리기 (round 아님)
- 증감 비율 수식 — `× (1 + 변화율/100)`
- 함수 정의·호출 구조, 빈칸에 함수 매칭하는 사고법
- 모듈 import 개념 (sqrt는 math 모듈에 있음)
- 수학의 `=` vs 코드의 `=` 의미 차이

### 자료형 의식의 중요성
- "10" (문자열) vs 10 (숫자) — 따옴표가 자료형 신호
- 자료형에 따라 같은 연산자도 동작 다름

## 막힌 부분
