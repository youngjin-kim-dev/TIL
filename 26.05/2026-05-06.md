## 배운 점

### SQL
- WHERE vs HAVING 차이
  - WHERE: 그룹화 **전** 필터링 (개별 행)
  - HAVING: 그룹화 **후** 필터링 (집계 결과)
- 집계 함수(COUNT, SUM, AVG 등)로 필터링 — 항상 HAVING 사용
- SQL 실행 순서: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

### GROUP BY 다중 컬럼
\`\`\`sql
GROUP BY USER_ID, PRODUCT_ID
\`\`\`
두 컬럼 조합으로 그룹화 — "같은 회원이 같은 상품 재구매" 같은 패턴.

### COUNT(*) — 그룹 내 행 개수
재구매, 중복 같은 패턴은 — `HAVING COUNT(*) > 1`로 표현.

## 막힌 부분
- "재구매"를 어떻게 SQL로 표현할지 처음에 고민 (HAVING 도달까지)
- WHERE에 COUNT 쓰면 안 된다는 거 인지
