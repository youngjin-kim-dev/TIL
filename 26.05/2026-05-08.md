## 배운 점

### SQL
- LIMIT 함수 — 결과의 상위 N개 반환
- "가장 ~한" 패턴 두 가지: ORDER BY + LIMIT 1, 서브쿼리 + MIN/MAX
- HAVING의 정확한 사용법 — 비교 조건이 와야 함 (값만 적으면 안 됨)
- GROUP BY 사용 시점 — 그룹별 집계 필요할 때만, 1개 행 찾기엔 불필요
- `!=` 연산자 — 같지 않음 (또는 `<>`)

### Python — 입출력 기본기
- `input()` 함수의 진짜 의미 — 사용자로부터 입력 받기
- `input(안내문)` — 안내 문구지 출력값 아님
- `print()`의 표준 입출력 패턴

### Python — f-string (가장 중요한 학습)
- `f'문자열 {변수}'` — 변수 값을 문자열에 삽입
- `f` 유무 차이 — 있으면 변수 처리, 없으면 그냥 글자
- `{}` 안에 — 변수, 수식, 메소드 다 가능

### Python — print()의 동작 이해
- `,`로 인자 구분 시 — 자동으로 공백 추가
- `+` 연결은 공백 없음, 자료형 변환 수동 (str())
- 출력 형식 정확히 맞추려면 — f-string이 표준

### Python — 입력 처리 패턴
- `map(함수, 반복가능)` — 각 원소에 함수 적용
- `strip()` — 양 끝 공백 제거
- `split()` vs `split(' ')` — 인자 없는 게 더 안전 (모든 공백 처리)
- 다중 할당 — `a, b = map(int, ...)`

## 막힌 부분
- "가장 먼저 들어온 동물" — GROUP BY + HAVING으로 시도, LIMIT 모름
- HAVING에 비교 조건 없이 `MIN(DATETIME)`만 적음
- `input('HelloWorld!')` — 안내문이 출력값인 줄 헷갈림 (시간초과)
- `print()`의 `,` 자동 공백으로 출력 형식 안 맞음

## 새로 학습한 패턴
- LIMIT N — 상위 N개 가져오기
- f-string — Python 표준 문자열 포매팅
- map() + split() + 다중 할당 — 표준 입력 처리
